# Quick Task 260615-u57: Xây dựng layout riêng cho trang Reviews Summary

Built a dedicated review layout system (schema extension + 4 shared components + 3 body layouts + dispatcher) and migrated the sample broker-comparison article to `reviewType: listicle`, rendering ranked broker cards with ratings, pros/cons, and CTAs.

## What Was Built

### Task 1: Extended content schema
- `src/content.config.ts`: added `ctaUrl` to `brokerSchema`; added `reviewType` (enum: company/comparison/listicle), `companyInfo`, `accountSteps`, `comparisonCriteria`, `winnerByCriteria`, `bestForScenarios`, `rankings` to the `articles` collection schema — all optional/defaulted, zero breakage to existing 50+ articles.

### Task 2: Shared review components
- `RatingBadge.astro` — numeric rating (e.g. "4.7") + 5-star row (rounded to nearest integer for fill), optional caption label.
- `ProsConsBox.astro` — two-column "Ưu điểm" / "Nhược điểm" grid, collapses to 1 column on mobile (≤580px), mint/peach accent backgrounds matching brand tokens.
- `ReviewCTA.astro` — bordered/offset-shadow button, `rel="noopener sponsored nofollow"` + `target="_blank"` on all outbound broker links; `sticky` prop fixes the CTA to the viewport bottom on mobile.
- `ReviewJumpNav.astro` — sticky horizontal anchor nav bar replacing the TOC sidebar for review pages.

### Task 3: Review body layouts + dispatcher
- `CompanyReviewBody.astro` — hero rating, "Thông tin nhanh" definition list, "Sản phẩm & dịch vụ" list, numbered "Hướng dẫn mở tài khoản" steps, pros/cons, sticky CTA.
- `ComparisonBody.astro` — comparison table (with winner-cell highlighting via `ComparisonTable`'s new `highlightMap` prop), per-broker pros/cons grid, "Nên chọn ai?" scenario list with per-scenario CTA to the winning broker.
- `ListicleBody.astro` — overview comparison table + ranked broker cards (rank badge, RatingBadge, badge tag, pros/cons, CTA), sorted by `rankings[].rank`.
- `ReviewLayout.astro` — mirrors `ArticleLayout`'s chrome (breadcrumb, schemas, header, hero, KeyTakeaways, FAQ, CitationBox, editorial-review, AuthorBox, ShareBar, RelatedArticles) but renders `ReviewJumpNav` instead of the TOC sidebar and dispatches to the appropriate body component based on `entry.data.reviewType`, falling back to plain `<slot />` when `reviewType` is absent.
- `ComparisonTable.astro` extended with optional `highlightMap?: Record<number, number[]>` prop for winner-cell styling (backward-compatible, defaults to `{}`).

### Task 4: Routing dispatch + sample migration
- `src/pages/[category]/[slug].astro` now renders `ReviewLayout` when `entry.data.layoutType === "review"`, otherwise `ArticleLayout` as before. Confirmed `dau-tu/[category]/[slug].astro` doesn't need the same change since `reviews` category group is `"Reviews"` (only handled by the non-dau-tu route).
- `review-cong-ty-chung-khoan-cho-nguoi-moi.md` migrated to `reviewType: listicle` with:
  - `rankings`: tcbs (#1, "Phí thấp nhất"), ssi (#2, "An toàn & uy tín nhất"), vps (#3, "Hỗ trợ môi giới tốt nhất"), vndirect (#4, "Giao diện thân thiện nhất")
  - `ctaUrl` per broker (tcbs.com.vn, ssi.com.vn, vps.com.vn, vndirect.com.vn)
  - `comparisonCriteria`: Phí giao dịch, Lãi suất margin, Đánh giá app
