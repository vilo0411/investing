# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Scope Note (read first)

Phase 2 is a **data-model/schema phase**, not a UI phase:

- Extends `src/content.config.ts` with optional fields (`citations`, `factCheckedDate`, `keyTakeaways`) — no rendering changes.
- Creates `src/data/authors.ts` as the single source of author/reviewer data.
- Refactors `AuthorBox.astro` and `site.author`/`site.authorProfile` usages (in `ArticleLayout.astro`, `BaseLayout.astro`) to read from `authors.ts`.

Per CONTEXT.md **D-06**, `AuthorBox.astro`'s **markup and styles must remain unchanged** — only its data source changes. No new components, layouts, pages, tokens, or visual treatments are introduced in this phase (those belong to Phase 3 and Phase 4).

This UI-SPEC therefore documents:
1. The **existing visual contract of `AuthorBox.astro`** that must be preserved byte-for-byte through the data-source refactor.
2. A minimal **"no visual change" regression checklist** to verify after the refactor.

No new design tokens, color roles, typography scale, or copywriting are defined here — Phase 1's token system (`src/styles/tokens/`) already covers everything `AuthorBox.astro` uses, and this phase must not introduce deviations from it.

## Verification Checklist — "No Visual Change" Regression Gate

This is the actionable contract for the executor and checker. After completing the `authors.ts` data-source refactor of `AuthorBox.astro`, `ArticleLayout.astro`, and `BaseLayout.astro`:

1. **DOM/markup diff**: Rendered HTML output of `AuthorBox.astro` on at least one article page must be structurally identical before/after the refactor (same elements, classes, nesting — only text node content sourced from `authors.ts` may differ if values changed, and per the Copywriting Contract above they should not differ).
2. **Computed styles**: `.author-box`, `.author-avatar`, `.author-name`, `.author-role`, `.author-bio`, `.author-expertise span`, `.author-profile-link`, `.author-label`, `.author-note` retain identical computed CSS (no new classes, no removed classes, no inline style changes).
3. **Text content**: Author name, role, bio, expertise tags, and note text render with the exact same strings as before the refactor (sourced now from `authors.ts` instead of `site.ts`/hardcoded JSX).
4. **JSON-LD (Person/Article)**: `BaseLayout.astro`/`ArticleLayout.astro` JSON-LD author fields (name, etc.) resolve to the same values as before, now sourced from `authors.ts`.
5. **Build integrity**: `astro check && astro build` succeeds for all 21 existing articles without requiring any of them to supply `citations`, `factCheckedDate`, or `keyTakeaways` (EEAT-01 acceptance criterion — these are schema/build-level checks, not visual, but block any visual verification if failing).
6. **Responsive check**: Mobile breakpoint (`@media (max-width: 560px)`) stacking behavior of `.author-box` / `.author-heading` is unchanged.

No screenshot diffing tooling is introduced in this phase (out of scope) — manual visual spot-check of one article page (desktop + mobile width) before and after the refactor is sufficient.
