## Spacing Scale

Declared values (already established in `src/styles/tokens/spacing.css`, reused as-is — values shown in px equivalents of the rem tokens):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon/label gaps, tight inline spacing |
| `--space-2` | 8px | Compact element spacing (badge padding, gap between trust-strip icon and label) |
| `--space-4` | 16px | Default element spacing (card internal gaps) |
| `--space-6` | 24px | Box padding (KeyTakeaways/CitationBox/Disclaimer/AuthorBox pattern, trust-strip card padding, about-page author preview card padding) |
| `--space-8` | 32px | Section-internal gaps, empty-state padding |
| `--space-12` | 48px | Major section breaks (hero grid gap, layout grid gaps) |
| `--space-16` | 64px | Page-level spacing (`.author-body` / `.section` vertical padding range) |
| `--space-24` | 96px | Reserved — not used in this phase |

Exceptions:
- Section vertical padding (`.section`, `.page-hero`, `.article-header`) uses literal px values (`44px`, `52px 0 34px`, etc.) following existing index.astro/about.astro convention — these are NOT on the 4px-multiple token scale strictly but ARE multiples of 4 (44, 52, 34 all divisible by 2 but check: 44✓, 52✓, 34✗). **New sections in this phase (trust strip, CategoryListing hero, about-page sections, disclaimer page) MUST use multiples of 4px** (e.g. 32px, 40px, 48px, 56px) for section padding — do not introduce new 34px/38px-style odd values.
- Touch targets for trust-strip links and topic-card links: minimum 44px tap height (existing `.topic-card`/`.start-item` already satisfy this via 20px padding + content).

## Typography

Reuses the Phase 1 type scale (`src/styles/tokens/typography.css`). No new sizes introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | `var(--text-body)` ≈ 16–18px (clamp) | 400 | 1.7 (`--leading-body`) |
| Label / eyebrow / badge | 0.78rem (≈12.5px) | 700–800 | 1.2, `text-transform: uppercase`, `letter-spacing: 0.08em` |
| Heading (H2/H3, section titles) | `var(--text-h3)` ≈ 18–24px (clamp) | 700 | 1.25–1.35 |
| Display (H1, hero/featured titles) | `var(--text-h1)` / `var(--text-h2)` ≈ 36–56px / 24–36px (clamp) | 700, `font-family: var(--font-serif)` | 1.2–1.25 |

Rules for this phase:
- Exactly **3 sizes** in active use for new elements: body (`--text-body`), label (0.78rem fixed), heading (`--text-h2`/`--text-h3` clamp range treated as one "heading" role).
- Exactly **2 weights**: 400 (body/regular) and 700 (semibold/bold — headings, labels, CTAs). The existing `font-weight: 800` on `.primary-action`/`.start-number` is a pre-existing exception; do not introduce new 800-weight usages in Phase 5 — use 700.
- New trust-strip item titles: 0.92rem, weight 700, line-height 1.4 (matches `.topic-card h3` pattern).
- New trust-strip item descriptions: 0.85rem, weight 400, color `var(--muted)`, line-height 1.55.
- `/disclaimer/` body copy: inherits `.prose` styles from `global.css` (body size, 1.7 line-height) — no new typography rules needed.
- About-page mission statement (D-02.1): treat as a `.lead`-style paragraph (existing `.lead` class in `global.css`) — larger body size, `var(--muted)` or `var(--text)` color.

## Color

Reuses Phase 1 tokens (`src/styles/tokens/colors.css`). No new colors introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` = `var(--color-neutral-50)` (#f7f8fa) page background; `var(--surface)` = `#ffffff` for cards/sections | Page background, all card/section surfaces (trust-strip cards, about preview card, CategoryListing hero/sidebar, disclaimer page body) |
| Secondary (30%) | `var(--surface-alt)` = `var(--color-navy-50)` (#eef3f9), `var(--color-neutral-100)` | Alternating section backgrounds (e.g. `.start-section` already uses `--color-neutral-100`), sidebar/latest-panel backgrounds, hover backgrounds (`.secondary-action:hover`) |
| Accent (10%) | `var(--color-accent)` = `var(--color-gold-700)` (#92660a) | RESERVED FOR: (1) section eyebrow/label text (`.eyebrow`, `.panel-label`, `.citation-label`, trust-strip eyebrow if used), (2) the small numeric/index markers (`.start-number` pattern) if reused in CategoryListing learning-path list. Accent is NOT used for buttons, links, borders, or backgrounds — those use `--brand`/`--color-brand-900`. |
| Destructive | `var(--destructive)` = `#dc2626` | Not used anywhere in Phase 5 (no destructive actions exist on homepage/category/trust pages — purely informational/navigational). Declared for contract completeness only. |

Primary brand color for interactive elements (links, primary CTA, borders-on-hover, active states): `var(--brand)` = `var(--color-brand-900)` = `var(--color-navy-800)` (#163050), with `var(--color-brand-700)`/`var(--color-brand-100)` for hover/tint states — this is the existing pattern across `.primary-action`, `.see-all`, `.featured-cta`, `.topic-card:hover`.

Accent reserved for: uppercase eyebrow/section labels (`.eyebrow`, `.panel-label`-style headings used in trust-strip section header, CategoryListing hero eyebrow, about-page section labels), and numbered list markers in learning-path style lists. Never used for body links, buttons, or card borders/backgrounds.

## Component & Layout Notes (project-specific addendum)

These notes translate the contract above into the concrete component/page work for this phase, anchored to D-01 through D-13 in `05-CONTEXT.md`.

### 1. Homepage trust strip (HOME-02, D-04–D-06)
- New `<section class="trust-strip">` placed immediately after `.home-hero`, before `.topic-section`.
- Layout: horizontal row of 3–4 cards, `display: grid; grid-template-columns: repeat(4, minmax(0, 1fr)); gap: var(--space-4);` collapsing to `repeat(2, 1fr)` at 860px and `1fr` at 640px (matches `.topic-grid` responsive pattern).
- Each card: `border: 1px solid var(--line); border-radius: var(--radius-md); background: var(--surface); padding: var(--space-6);` with a small accent-colored title (0.92rem, weight 700) and `var(--muted)` body (0.85rem). Whole card is an `<a>` (like `.topic-card`), hover state matches `.topic-card:hover` (border-color `--color-brand-100`, `box-shadow: var(--shadow-sm)`).
- Section header uses the standard `.eyebrow` + `<h2 class="section-title">` pattern already used by `.topic-section`/`.start-section`.

### 2. Shared `CategoryListing` component (CATG-01, D-07–D-09)
- Location: `src/components/CategoryListing.astro` (matches existing `src/components/*.astro` convention — D-07 discretion resolved toward `components/` since `ArticleList.astro`, `AuthorBox.astro`, `Breadcrumb.astro` all live there and this is a reusable presentational unit, not a layout shell).
- Props: `{ category: Category; articles: CollectionEntry<"articles">[] }`.
- Renders the entire body currently in `[category].astro` lines 53–115: page-hero (breadcrumb + eyebrow + h1 + lead + article-count badge), then `.section` with `.category-layout` grid (learning-path `ArticleList` + `.latest-panel` sidebar), or the empty-state card.
- D-08: replace the inline `<nav class="breadcrumb">` (lines 56–60) with `<Breadcrumb items={[{ label: "Trang chủ", href: "/" }, { label: category.title }]} />` — same 2-level pattern. For the `dau-tu/[category].astro` tree, if that route currently includes an intermediate "Đầu tư" breadcrumb level, use 3 items: `[{ label: "Trang chủ", href: "/" }, { label: "Đầu tư", href: "/dau-tu/" }, { label: category.title }]`. Both route files pass their own breadcrumb items array as a prop OR the component derives it from `category.group` — recommend passing `breadcrumbItems` as an explicit prop from each route file since the two trees differ in depth (keeps `CategoryListing` route-tree-agnostic).
- Both `[category].astro` and `dau-tu/[category].astro` reduce to: `getStaticPaths()` (unchanged filter logic, D-09) + article fetch/sort + `<CategoryListing category={category} articles={articles} breadcrumbItems={...} />`.
- Visual tokens: ensure `.article-count-badge`, `.empty-state`, `.latest-panel` etc. (currently already token-based per the file I read) remain unchanged — D-08's "ensure visual tokens match Phase 1" is largely already satisfied; the main visual delta is the Breadcrumb component swap.

### 3. `/about/` rewrite (TRST-01, D-01–D-03)
- Remove `.author-hero`/`.author-layout`/`.author-sidebar` full-dossier sections (those now live at `/author/[slug]/`).
- New page structure:
  1. **Hero**: eyebrow "Giới thiệu" + `<h1>` mission headline ("Vì sao ValueInvesting.com.vn ra đời") + `.lead` paragraph covering audience (người mới bắt đầu) and differentiation (giáo dục, có nguồn, không khuyến nghị). Use `.page-hero`-style section (reuse `global.css` `.article-header` or `[category].astro`'s `.page-hero` pattern — pick whichever is closer; both already token-driven).
  2. **Editorial process summary**: 2–3 step list (numbered, can reuse `.start-item`/`.start-number` visual pattern from index.astro for a "01/02/03" step list), ending with a CTA link "Xem chính sách biên tập đầy đủ" → `/editorial-policy/`.
  3. **Author preview card**: small card — `.profile-card`-style box (`border: 1px solid var(--line); border-radius: var(--radius-md); background: var(--surface); padding: var(--space-6);`) containing avatar-initial circle (smaller version of `.author-avatar-large`, e.g. 64px), `author.name`, `author.role`, one-line `author.bio` (truncate via `-webkit-line-clamp: 2` if needed), and CTA "Xem hồ sơ đầy đủ" → `/author/{author.slug}/`. Source from `src/data/authors.ts` (D-03).
  4. **Site stats**: reuse `.stat-stack` markup/styling from current about.astro but retitle the section "Quy mô nội dung" and remove the "Năm cập nhật hồ sơ" stat if it reads as profile-specific (keep articleCount + categoryCount at minimum).
- All new sections use the `.section`/`.container` wrapper pattern from `global.css`/index.astro.
