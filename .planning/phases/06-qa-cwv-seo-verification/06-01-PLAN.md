  <name>Task 1: Build the Node.js verification script (QASE-02, QASE-03, QASE-04, D-03, D-04, D-06, D-07)</name>
  <files>scripts/run-verification.mjs</files>
  <read_first>
    astro.config.mjs (to extract sitemap and redirect configurations)
    .planning/phases/06-qa-cwv-seo-verification/06-RESEARCH.md (for verification specifications and patterns)
  </read_first>
  <action>
    Create a standalone Node.js ES module file `scripts/run-verification.mjs`. The script must load no external npm packages (only native `fs`, `path`, `url`, etc.) and do the following:
    
    1. Parse build output directory `dist/` (recursively scanning all `.html` and `.css` files).
    
    2. CWV Auditor (D-03):
       - Scan all `<img>` tags in HTML files to assert that 100% of them have both `width` and `height` attributes (prevent CLS). Log details of any violating image (path, tag text).
       - Scan all CSS files (including CSS files inside `dist/_astro/`) to verify that all `@font-face` declarations contain `font-display: swap;`.
       
    3. Structured Data Validator (D-04):
       - Extract `<script type="application/ld+json">` contents from HTML files using regex.
       - Parse the JSON blocks and validate compliance for core schemas:
         - `Article` (must have `headline`, `author` with type `Person` and `name`, `publisher` with type `Organization`, `datePublished`, `dateModified`).
         - `Person` (author profile pages must have `name`, and optionally check for fields like `sameAs` or credentials).
         - `BreadcrumbList` (must have `itemListElement` containing `position`, `name`, `item` per item).
         - `WebSite` / `Organization` / `FAQPage` if present.
       
    4. Sitemap & URL Freeze Auditor (D-06, D-07):
       - Read and parse `dist/sitemap-0.xml` and diff it against the list of 38 expected baseline URLs (derived from original pages). Assert that no baseline URLs are missing or renamed.
       - Crawl all internal anchor links (`<a href="...">`) in the built HTML files. Verify that each internal link points to a target file that actually exists on disk (e.g. `/dau-tu/co-phieu/` maps to `dist/dau-tu/co-phieu/index.html`). Correctly ignore external URLs (beginning with http/https pointing outside `valueinvesting.com.vn`), javascript links, and mailto/tel anchors. Strip hashes (`#`) when checking path existence on disk.
       - Read redirects configuration from `astro.config.mjs` and verify that for each redirect path, Astro has generated a redirect HTML file in `dist/` containing `<meta http-equiv="refresh" content="0;url=...">` and that the destination target is a valid, existing HTML file.
       
    5. Consolidate results:
       - The script should return exit code 0 if all assertions pass, and non-zero exit code if there are any blocker violations. It must write a summary report containing counts of checks passed/failed to `dist/verification-report.json`.
  </action>
  <acceptance_criteria>
    - scripts/run-verification.mjs exists and parses without syntax errors under Node.js
    - running `node scripts/run-verification.mjs` does not require npm install of external packages
    - script has defined checks for `<img>` width/height attributes, `@font-face` font-display, JSON-LD Article validation, sitemap parsing, and internal link resolution
  </acceptance_criteria>
  <verify>
    <automated>node -c scripts/run-verification.mjs</automated>
