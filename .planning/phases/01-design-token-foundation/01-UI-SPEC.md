# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Phase nature:** This is a CSS-token-only phase. No new pages, components, or markup are introduced (DSGN-01–05). The contract below locks the actual token VALUES that `src/styles/tokens/*.css` must define and that `aliases-legacy.css` must map old names onto. The planner/executor should treat every hex/clamp/duration value in this document as the literal value to write into the token files — not a starting point for further exploration.

## Token Architecture (DSGN-01, D-09, D-12)

```
src/styles/
├── global.css              # @import tokens + aliases (in this order), then existing reset/component CSS unchanged
└── tokens/
    ├── colors.css           # primitives (--color-navy-50..900, --color-gold-50..900,
    │                         #   --color-neutral-50..900) + NEW semantics (--brand, --text, --bg, etc.)
    ├── typography.css        # --font-serif/--font-sans + clamp() type scale + --article
    ├── spacing.css            # --space-1..--space-24 (values UNCHANGED, just relocated)
    ├── effects.css            # --radius-*, --shadow-*, --transition-fast/base/slow
    └── aliases-legacy.css     # OLD primitive/alias-only names → NEW tokens (DSGN-02)
```

**Import order in `global.css` (critical — see RESEARCH.md Pitfall 1):**
```css
@import "./tokens/colors.css";
@import "./tokens/typography.css";
@import "./tokens/spacing.css";
@import "./tokens/effects.css";
@import "./tokens/aliases-legacy.css";
/* existing reset + component rules below, UNCHANGED except --transition rename note */
```

Rule: NEW semantic names (`--brand`, `--text`, `--bg`, `--surface`, `--surface-alt`, `--muted`, `--line`, `--accent`, `--brand-strong`, `--font-serif`, `--font-sans`, `--article`, `--max`) are declared **exactly once**, in their primitive/typography/spacing files. `aliases-legacy.css` contains ONLY names that do not already exist as new semantics (legacy-only primitives: `--color-brand-900/700/100`, `--color-neutral-950/700/400/100/0`, `--color-accent`, `--color-accent-soft`, `--transition`).

### Primitive scale: Gold/Premium Accent (`--color-gold-50..900`) — D-04

| Token | Hex | Usage |
|-------|-----|-------|
| `--color-gold-50` | `#fdf8ec` | Badge background tint ("Editor's pick") |
| `--color-gold-100` | `#faedc7` | Hover background for gold elements |
| `--color-gold-300` | `#edcb7a` | Decorative borders |
| `--color-gold-500` | `#c9952c` | Default gold accent — icons, borders, badge backgrounds (NOT body text — only ~2.8:1 on white) |
| `--color-gold-600` | `#ad7d1f` | (reserved) |

**WARNING — hard rule for executor:** `--color-gold-500` MUST NOT be used for body/label text directly on white/light backgrounds (fails AA at ~2.8:1). Use only for: icon fills, borders, badge background fill (with dark `--color-gold-900` or `--color-navy-900` text on top), or decorative large text ≥24px bold (AA-large 3:1 threshold). Any gold *text* on light background → use `--color-gold-700` or `--color-gold-900`.

### Legacy Green (`#006b2c`) — placement decision (resolves Open Question 1)

**Decision:** Demote to alias-only legacy token. Do NOT promote into the new primitive palette (avoids navy + gold + green competing as three "primary" brand colors, per research recommendation). Defined only in `aliases-legacy.css`:

```css
--color-green-700: #006b2c;  /* AA: 6.69:1 on white — kept for reference, not promoted */
--color-green-50:  #eafaf0;
```

`--color-accent` and `--color-accent-soft` (legacy names, used for `.eyebrow` text, footer link hover, header top accent gradient) map to **gold**, not green:
```css
--color-accent: var(--color-gold-500);       /* .eyebrow text, footer link hover */
--color-accent-soft: var(--color-gold-50);   /* unused currently but kept for compat */
```
Rationale: `.eyebrow` and footer-link-hover are small accent touches; gold reads as "premium/trust" per D-04 and replaces green's role cleanly. If DSGN-05 visual audit reveals `.eyebrow` text at `--color-gold-500` is illegible (it is — only 2.8:1), the alias must instead point `--color-accent` to `--color-gold-700` (5.09:1, AA pass) for text contexts. **Executor instruction:** since `.eyebrow` renders gold-colored TEXT (`color: var(--accent)`), use `--color-gold-700` for `--color-accent`, NOT `--color-gold-500`. Update the example above accordingly:
```css
--color-accent: var(--color-gold-700);  /* text usage — AA pass at 5.09:1 */
--color-accent-soft: var(--color-gold-50);
```

### New Semantic Color Tokens (defined once, in `colors.css`)

| Token | Value | Usage (60/30/10 mapping) |
|-------|-------|---------------------------|
| `--bg` (60% dominant) | `var(--color-neutral-50)` `#f7f8fa` | Page background (`html`) |
| `--surface` (30% secondary) | `#ffffff` | Cards, header, nav search box, page-hero, article-header |
| `--surface-alt` (30% secondary) | `var(--color-navy-50)` `#eef3f9` | `.band` sections, sidebar tint |
| `--text` | `var(--color-neutral-900)` `#1a2330` | Primary body/heading text |
| `--muted` | `var(--color-neutral-700)` `#475569` | Secondary text, `.meta`, `.lead`, captions |
| `--line` | `var(--color-neutral-100)` `#eef1f5` | Borders, dividers, `.card` border |
| `--brand` (10% accent role 1) | `var(--color-navy-800)` `#163050` | Headings (h1-h3), brand wordmark, nav links active state, blockquote border |
| `--brand-strong` | `var(--color-navy-600)` `#2c5282` | Link hover state |
| `--accent` (10% accent role 2 — gold) | `var(--color-gold-700)` `#92660a` | `.eyebrow` text, footer link hover ONLY |

**Accent reserved for:** `.eyebrow` labels (category/section eyebrow text), footer column link `:hover` color, header top gradient accent line (3rd stop), and future "Editor's pick"/premium badges (Phase 3+, gold-50/gold-500/gold-700 combo). Do NOT apply `--accent` to general buttons, primary CTAs, or body links — those use `--brand`/`--brand-strong`.

**Destructive color:** Not applicable this phase — no destructive actions exist in a token-only phase. Reserve `--color-red-600: #dc2626` (standard, AA-compliant on white at ~4.8:1) as a placeholder semantic `--destructive` token for future phases (Phase 3+ may need it for "delete"/error states), but it is NOT required to be wired into any component this phase. If added, define in `colors.css`: `--destructive: #dc2626;`

## Typography (DSGN-03, DSGN-04, D-05–D-08, D-08a)

### Font Stack

| Role | Font | Weights | Vietnamese coverage |
|------|------|---------|---------------------|
| `--font-serif` | `'Source Serif 4', Georgia, serif` | 400, 500, 700 | Verified — full `U+1EA0-1EF9` block + `U+0102-01B0` + `U+20AB` at all 3 weights |
| `--font-sans` | `'Inter', ui-sans-serif, system-ui, -apple-system, sans-serif` | 400, 500, 700 | Verified — same coverage |

**Google Fonts `<link>` replacement in `BaseLayout.astro`** (replaces DM Serif Display + DM Sans block, lines 55-61):
```html
<!-- Google Fonts: Source Serif 4 + Inter -->
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=Source+Serif+4:wght@400;500;700&family=Inter:wght@400;500;700&display=swap"
  rel="stylesheet"
/>
```
(Non-variable weight-list form — verified to return identical Vietnamese `@font-face` blocks; simpler than the `opsz` variable-axis form.)

**Font-weight constraint (D-08, Pitfall 4):** Only 400/500/700 may be used anywhere in `.astro` `<style>` blocks. `.mega-item-title` currently uses `font-weight: 600` — executor MUST change this to `500` (closer visual match, avoids browser font-weight synthesis). This is the only known 600-weight usage (RESEARCH.md A4); if `astro check`/grep finds others, apply the same fix (round to nearest of 400/500/700).

| Role | Font | Weight | Line Height |

### Article Content Width (DSGN-03, Pitfall 3)

`--article` token changes from `760px` to `65ch`:
```css
--article: 65ch;
```
**Executor verification step (mandatory, not optional):** After implementing, measure rendered `.article-container` width at 375px, 768px, 1280px, 1920px viewports in `npm run preview`. Target range: 680-760px at desktop widths (equivalent to the visual feel of the old 760px). If `65ch` in Inter renders outside ~650-780px at 1280px+ viewport, fall back to `clamp(60ch, 70ch, 760px)` or a fixed `760px` with an inline comment `/* ~65ch in Inter */` — DSGN-03's literal requirement is "~65ch", a small documented deviation for visual continuity is acceptable.
