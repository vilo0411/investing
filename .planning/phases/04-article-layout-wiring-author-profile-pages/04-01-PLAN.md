    Create `scripts/backfill-key-takeaways.mjs` as an ES module (project has `"type": "module"` in package.json) that:
    - Iterates all `.md` files in `src/content/articles/` via `fs.readdirSync`
    - For each file, parses with `matter(raw)` from `gray-matter`
    - Matches the section via regex `/^## Key takeaways\s*\n((?:- .+\n?)+)\n*/m` against `parsed.content`
    - If no match: `console.warn("SKIP (no Key takeaways section found): " + file)` and continue (fail loudly per RESEARCH.md, do not silently skip without logging)
    - Extracts bullets: split captured group on `\n`, filter lines starting with `- `, strip the `- ` prefix and trim
    - Sets `parsed.data.keyTakeaways = bullets` (array of strings, verbatim text — no wording changes)
    - Removes the matched section from `parsed.content` via `.replace(sectionRegex, "")`, then trims leading newlines via `.replace(/^\n+/, "")`
    - Writes back with `matter.stringify(parsed, parsed.data, { lineWidth: -1 })` to `fs.writeFileSync` (the `{ lineWidth: -1 }` option prevents js-yaml from re-wrapping long Vietnamese strings in unrelated fields like `description`/`sources`, per Pitfall 2)
    - Logs `console.log("Updated " + file + ": " + bullets.length + " key takeaways")` per file, and a final summary line counting total files processed vs skipped

    Run the script via `node scripts/backfill-key-takeaways.mjs` from the project root. Verify the console output shows 21/21 files processed with non-zero bullet counts (each article has at least one bullet per RESEARCH.md's grep confirmation).
  </action>
  <acceptance_criteria>
