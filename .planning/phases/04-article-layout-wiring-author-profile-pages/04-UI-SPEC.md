## Typography

All values from Phase 1 (`src/styles/tokens/typography.css`) and Phase 3 component styles, reused verbatim. This phase introduces **no new font sizes or weights**.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|

**Declared scale for this phase (2 weights only, as required):**
- 400 (regular) ‚Äî body text, bio paragraphs, list items, citation/source links, breadcrumb links
- 700 (bold) ‚Äî eyebrow labels, H1/H2 headings, avatar initial, profile-card headings

Font sizes used (3, within 3‚Äì4 range): `0.78rem`‚Äì`0.88rem` (labels/eyebrows), `0.9rem`‚Äì`1.05rem` (body/bio/quote), `clamp(1.45rem, 2.4vw, 4.3rem)` (H1/H2 headings, fluid). Quote-block uses `1.05rem` at line-height 1.8 (matches `about.astro` `.quote-block` pattern).

Body line-height: 1.7 (bio sections) / 1.6 (callout box lists, AuthorBox/CitationBox/KeyTakeaways body text ‚Äî Phase 3 convention).

## Component-Specific Visual Contracts

### 1. `ArticleLayout.astro` ‚Äî composition reorder (no new components, D-09/D-10/D-11)

Render order inside `<article class="prose article-content">` (the content column of the existing 2-column `.article-body--toc` grid):

```
<Breadcrumb items={breadcrumbItems} />        <!-- replaces inline <nav class="breadcrumb"> -->
<header> ... title/lead/meta/trust strip ... </header>  <!-- unchanged -->
<div class="article-body--toc">
  <article class="prose article-content">
    [<Disclaimer />]                          <!-- optional consolidation of .ymyl-note, Claude's discretion -->
    <KeyTakeaways items={entry.data.keyTakeaways} />   <!-- NEW position: first child, before <slot/> -->
    <slot />                                   <!-- article body, "## Key takeaways" section removed -->
    {FAQ section ‚Äî unchanged}
    <CitationBox sources={entry.data.sources} updatedDate={updatedDate} />  <!-- replaces .source-section -->
    <AuthorBox />                              <!-- existing, links now resolve to /author/{slug} -->
    <ShareBar />
    <RelatedArticles />
  </article>
  <aside class="toc-sidebar"><TOC /></aside>
</div>
```

- **KeyTakeaways placement (D-09):** stays inside `.article-content` (content column only), NOT full-width above the 2-column grid. On mobile (`@media (max-width: 1080px)`, existing breakpoint), the grid collapses to 1 column and KeyTakeaways naturally appears above body content with TOC hidden below ‚Äî no new media query needed.
- **Breadcrumb swap (D-10):** delete the old `.breadcrumb`, `.breadcrumb a`, `.breadcrumb a:hover`, `.breadcrumb span[aria-current]` scoped CSS rules (current lines ~191-214) ‚Äî `Breadcrumb.astro` ships its own scoped `<ol>/<li>` styles. No visual regression expected; `Breadcrumb.astro`'s styles were already verified against the design tokens in Phase 3.
- **CitationBox swap (D-11):** delete `.source-section` CSS rules, but carefully split any selectors shared with `.editorial-review` (current lines ~260-267, 288-292) ‚Äî `.editorial-review` styling must remain intact. The `.source-section`'s "xem ch√≠nh s√°ch ngu·ªìn tham kh·∫£o" sentence (link to `/sources-policy/`) is dropped ‚Äî `CitationBox` has no slot for it, and `/sources-policy/` remains linked from `/editorial-policy/` and the site footer. This is an accepted minor content reduction per D-11's "replacing" intent.
- **Disclaimer consolidation (Claude's discretion):** if `.ymyl-note` ‚Üí `<Disclaimer />` is adopted, the heading text changes from "L∆∞u √Ω tr∆∞·ªõc khi ƒë·ªçc:" to "L∆∞u √Ω quan tr·ªçng:" (Disclaimer.astro's hardcoded heading). This is an accepted minor copy change ‚Äî both convey the same YMYL disclosure; body text (`site.disclosure`) is identical either way. If exact heading-copy preservation is preferred, skip consolidation and leave `.ymyl-note` as-is ‚Äî both are valid outcomes of this contract.
- **JSON-LD `author` expansion (D-08):** see Copywriting Contract above. No new visual surface ‚Äî JSON-LD is invisible, inline `<script type="application/ld+json">`.
